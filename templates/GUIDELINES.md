## Database Conventions

- **Engine:** InnoDB
- **Charset:** utf8mb4
- **Public IDs:** UUIDs (not auto-increment) for user-facing models
- **Soft deletes:** All user-facing data (users, posts, orders)
- **Timestamps:** Every table has `created_at` and `updated_at`
- **Migrations:** Immutable once committed — always create new migrations for changes
- **Indexes:** Index every foreign key and every column used in WHERE/ORDER BY
- **Naming:** `snake_case` for column names, plural for table names

### Table Naming Pattern
- `users` — regular plural
- `post_user` — pivot tables (alphabetical order)
- `password_reset_tokens` — clear purpose

## Security

### Authentication
- **Driver:** Sanctum (stateless)
- **Token expiry:** 15 minutes (access), 7 days (refresh)
- **Password policy:** Minimum 12 characters, 1 uppercase, 1 number, 1 special

### Rate Limiting
| Endpoint | Limit | Window |
|----------|-------|--------|
| Login | 5 attempts | 1 minute per IP+email |
| Registration | 3 attempts | 1 hour per IP |
| Password reset | 3 attempts | 1 hour per email |
| General API | 100 requests | 1 minute per token |

### CORS
- Allowed origins: `https://example.com`, `https://admin.example.com`
- Methods: GET, POST, PUT, PATCH, DELETE
- Credentials: true

## Testing

- **Framework:** Pest / PHPUnit
- **Database:** SQLite in-memory for tests (with migrations)
- **Factories:** Define factories for all models (use realistic data)
- **Coverage target:** minimum 80%
- **Test types:**
  - `tests/Unit/` — service/repository logic in isolation
  - `tests/Feature/` — API endpoint integration tests
  - `tests/Unit/Services/` — one test file per service
  - `tests/Feature/Api/V1/` — one test file per resource
