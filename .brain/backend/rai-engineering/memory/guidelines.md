## Architecture

```
Pattern: Service Layer + Repository
├── Controllers — thin, handle HTTP only (max 30 lines per method)
├── Services — business logic, stateless, testable, transactional
├── Models — Eloquent/ORM, relationships, scopes, UUIDs
└── API Resources — response transformation, field filtering
```

**Key patterns:**
- Controllers delegate to Services (never direct DB calls in controllers)
- Services use Repositories for data access
- Validation uses Form Request classes (not inline in controllers)
- Responses use API Resource classes (not returning models directly)

## Database Conventions

- **Engine:** InnoDB
- **Charset:** utf8mb4
- **Public IDs:** UUIDs (not auto-increment)
- **Soft deletes:** All user-facing data
- **Timestamps:** Every table has `created_at` and `updated_at`
- **Indexes:** Every foreign key + every WHERE/ORDER BY column
- **Naming:** `snake_case` columns, plural table names
